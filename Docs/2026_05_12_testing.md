# Behat coverage plan — `Sandstorm.NodeTypes.Folder`

Date: 2026-05-12
Scope: full behavioral coverage for everything this package ships.
Approach: **end-to-end through the real router + projection + catch-up hook + command-bus**, never mocks of production internals. Every learning below is lifted from `Packages/Application/Neos.Neos/Tests/Behavior/Features/FrontendRouting/` and its bootstrap.

## 0. Why this plan exists

The package overrides `Neos.Neos:DocumentUriPathProjection` with a folder-aware fork, adds a write-side collision defense (`UriCollisionCommandHook` + `UriCollisionCheck`), a router-cache catch-up hook, and a `HideUriSegment` mixin. The bootstrap (`Tests/Behavior/Features/Bootstrap/FeatureContext.php`, `behat.yml.dist`, `Configuration/Testing/Behat/Settings.ContentRepositoryRegistry.yaml`) already exists but **zero `.feature` files are checked in**.

Per `CLAUDE.md`: assert observable behavior (URI paths, projection rows, routing results, command-rejection), never internal strings; reuse `neos/contentrepository-testsuite` traits, never duplicate production wiring.

## 1. Learnings from Neos.Neos FrontendRouting tests (load-bearing reference)

All step phrases below are **literal** — paste-ready in `.feature` files. File:line refs are into `Packages/Application/Neos.Neos/Tests/Behavior/`.

### 1.1 The HTTP layer is REAL, not stubbed

`Features/Bootstrap/RoutingTrait.php:142-153` — `@When I am on URL :url` sets the URI on Flow's `FunctionalTestRequestHandler`; `@Then the matched node should be …` (`:156`) and `@Then The URL :url should match the node …` (`:185`) actually invoke `$router->route($routeContext)` with `SiteDetectionMiddleware` populating route parameters. `@Then The node … should resolve to URL …` (`:223`) uses the real Flow URI builder. `BrowserTrait::setupBrowserForEveryScenario` (`:52-67`) wires `CustomizedInternalRequestEngine` + `FunctionalTestRequestHandler` per scenario.

**Consequence for us:** we test the *router output* the user actually sees — `/child` vs `/folder-a/child` — not "the projection wrote column X". The projection rows are a secondary assertion (see 1.3).

### 1.2 Site fixture is in-memory Doctrine `postLoad` injection

`RoutingTrait.php:109-139` — `@Given the sites configuration is: """yaml…"""` registers a Doctrine `postLoad` listener that injects `sitesConfiguration` into every loaded `Site` entity. Combined with `@Given A site exists for node name :nodeName` (`:79`) this gives a fully resolvable Site without writing Settings files. Site config snippet (literal, from `Basic.feature:55-66`):

```yaml
Neos:
  Neos:
    sites:
      'node1':
        preset: 'default'
        uriPathSuffix: ''
        contentDimensions:
          resolver:
            factoryClassName: Neos\Neos\FrontendRouting\DimensionResolution\Resolver\NoopResolverFactory
```

For dimensions tests, swap `NoopResolverFactory` for `UriPathResolverFactory` (per `Dimensions.feature`).

### 1.3 Projection rows are assertable as-is — `hideurisegment` slots in for free

`RoutingTrait.php:255-272` — `@Then I expect the documenturipath table to contain exactly:` accepts a `TableNode` whose **headers are column names** and **values are JSON-decoded** (`JSON_THROW_ON_ERROR`). The SELECT is built from the headers:

```php
$columns = implode(', ', array_keys($expectedRows->getHash()[0]));
$actualResult = $dbal->fetchAllAssociative(
  'SELECT ' . $columns . ' FROM ' . $tablePrefix . '_uri ORDER BY nodeaggregateidpath, dimensionspacepointhash'
);
```

**This means we get `hideurisegment` assertions for free** — just include the column in the header row. **No new step or trait required.** This kills the `HideUriSegmentRoutingTrait` proposal from the previous draft of this plan.

Asseratble columns (per `Lowlevel_ProjectionTests.feature:57-63` + the folder schema builder we add): `uripath`, `nodeaggregateidpath`, `nodeaggregateid`, `parentnodeaggregateid`, `precedingnodeaggregateid`, `succeedingnodeaggregateid`, `nodetypename`, `dimensionspacepointhash`, **`hideurisegment`** (new, from `FolderAwareDocumentUriPathSchemaBuilder`).

### 1.4 Subscription replay is a one-liner step

`CRTestSuiteTrait::iReplayTheProjection()` — `@When I replay the :projectionName projection` (used in `Basic.feature:135`). Invokes the real `ContentRepositoryMaintainer::replaySubscription()`. This is exactly the thing the deploy procedure runs (`./flow subscription:replay`), so we can E2E-test replay correctness.

### 1.5 Exception assertions are by **short name**

`ExceptionsTrait.php:40-57` — `@Then an exception of type :expectedShortExceptionName should be thrown [with code :code | with message: """…"""]`. Uses `ReflectionClass::getShortName()`, so we assert `"UriPathCollisionDetected"` (not the FQN). The throwing command must be wrapped by an existing step that funnels through `tryCatchingExceptions()` — `CRTestSuiteTrait` does this for the standard command-execution steps.

### 1.6 Tags `@flowEntities @contentrepository` at Feature level are load-bearing

`@flowEntities` triggers `BrowserTrait`'s `@BeforeScenario` (HTTP request handler init). `@contentrepository` keys CR setup. Every feature file we author starts with these two tags.

### 1.7 Dimension tests use a literal `using the following content dimensions:` table

`Dimensions.feature:5-8`:

```gherkin
Given using the following content dimensions:
  | Identifier | Values      | Generalizations |
  | market     | DE, CH      | CH->DE          |
  | language   | en, de, gsw | gsw->de->en     |
```

Then `I am in workspace "live" and dimension space point {"market":"DE", "language":"en"}` and DSPs are JSON throughout.

### 1.8 Command payloads — exact column names

- `CreateRootWorkspace`: `workspaceName`, `newContentStreamId`.
- `CreateRootNodeAggregateWithNode`: `nodeAggregateId`, `nodeTypeName`.
- `CreateNodeAggregateWithNode` (batched): `nodeAggregateId | parentNodeAggregateId | nodeTypeName | initialPropertyValues | nodeName`.
- `SetNodeProperties`: `nodeAggregateId`, `originDimensionSpacePoint`, `propertyValues` (JSON object).
- `MoveNodeAggregate`: `nodeAggregateId`, `dimensionSpacePoint`, `newParentNodeAggregateId`, `newSucceedingSiblingNodeAggregateId`.
- `CreateNodeVariant`: `nodeAggregateId`, `sourceOrigin`, `targetOrigin` (both JSON DSPs).
- `TagSubtree` / `UntagSubtree`: `nodeAggregateId`, `coveredDimensionSpacePoint`, `nodeVariantSelectionStrategy`, `tag`.

### 1.9 Bidirectional routing assertions are the gold standard

`Basic.feature:68-86` does **both** "I am on URL X → matched node is Y" **and** "node Y in dimension D should resolve to URL X" for every fixture position. We mirror that pattern for every Folder-transparency scenario. Asymmetry is a bug class on its own (the router and the URI builder are different code paths through the same projection rows).

### 1.10 Cache gotcha — replay won't behave unless `DocumentUriPathFinder` cache is disabled

`Basic.feature:98-106` comment block — bug #4253 was that cache survived across scenarios. Pattern: when a scenario does a `replay`, the surrounding scenario must either pre-disable the cache or accept "cache-warm" results. We follow the upstream convention (the testsuite trait handles this; we do not roll our own).

### 1.11 Existing trait composition required to make all of the above work

`Neos.Neos`' `FeatureContext.php:30-56` mixes:

```

CRTestSuiteTrait { deserializePropertiesCrTestSuiteTrait; },
CRBehavioralTestsSubjectProvider, RoutingTrait, MigrationsTrait,
```

Everything else stays out unless a specific scenario needs it.

## 2. Test architecture (revised, E2E-first)

### 2.1 Real SUT through the whole stack

- **HTTP**: `@When I am on URL …` → real Flow router → real `DocumentUriPathProjection` (our fork) rows → real `SiteDetectionMiddleware`. No browser stub, no router stub.
- **Commands**: `@When the command X is executed with payload:` → real command bus → real `UriCollisionCommandHook` (Defense A) → real `UriCollisionCheck` queries against real DBAL → real exception path captured by `ExceptionsTrait`.
- **Projection**: real `cr:setup` brings up the schema including our `hideurisegment` column via `FolderAwareDocumentUriPathSchemaBuilder`. Real subscription catch-up applies events.
- **Catch-up hook**: real `FolderRouterCacheHook` → real `RouterCachingService`. For assertion, we recycle Neos.Neos' existing "flush tag recording" mechanism used by `RouteCache.feature` (still TBD which trait — see §6.2).
- **Replay**: real `replaySubscription()` via `@When I replay the "Neos.Neos:DocumentUriPathProjection" projection`.

### 2.2 What we deliberately do NOT mock

- No `DocumentUriPathFinder` mock — read the real projection.
- No `Connection`/`DBAL` mock — use the real Doctrine connection.
- No fake `RouterCachingService` of our own — recycle the upstream one or skip the assertion and fall back to "does the next URL resolve right after the toggle".
- No `NodeTypeManager` mock — the design explicitly removed runtime NodeTypeManager reads from the projection; tests should encode this by asserting the `hideurisegment` column is correct **without** depending on NodeTypeManager queries.

### 2.3 Tree fixtures (named, reused, grep-able)

**Tree A — single dimension, one folder layer.** Used by 3.2, 3.4, 3.6, 3.11, 3.12.

```
sites/node1 (segment unused)
├─ folder-a   (Document.Folder, hide=true)            segment="folder-a"
│   └─ child-in-folder  (Test.Routing.Page)           segment="child"
└─ sibling-of-folder    (Test.Routing.Page)           segment="sibling"
```

Expected URLs (folder transparent): `/sibling`, `/child`. Folder itself has no own URL (its row stores `uripath="folder-a"` for descendant rewrites but is not routed to).

**Tree B — two-level nesting + opaque folder for contrast.** Used by 3.3, 3.5.

```
sites/node1
├─ folder-outer            (Folder, hide=true,  segment "folder-outer")
│   ├─ folder-inner-transparent (Folder, hide=true,  segment "folder-inner-transparent")
│   │   └─ deep-child          (Page,   segment "deep")
│   └─ folder-inner-opaque (Folder, hide=false, segment "visible-folder")
│       └─ deep-child-2       (Page,   segment "deep")
└─ regular-page             (Page,   segment "regular")
```

Expected: `/deep` (transparent chain), `/visible-folder/deep` (opaque folder participates).

**Tree C — multi-dimension (DE/EN).** Used by 3.7.

Tree A in DE; EN created via `CreateNodeVariant`.

### 3.3 `FrontendRouting/NestedFolders.feature` — Tree B, bidirectional

- `deep-child` resolves to `/deep` AND `/deep` matches `deep-child`.
- `deep-child-2` resolves to `/visible-folder/deep` AND `/visible-folder/deep` matches `deep-child-2`.
- Rename `folder-inner-opaque.uriPathSegment` to `nu` via `SetNodeProperties` → `deep-child-2` now resolves to `/nu/deep` (covers the *cascading* core UPDATE branch in hunk 4 of the projection).
- Rename `folder-outer.uriPathSegment` → no descendant URL changes (covers the *folder-row-only* UPDATE branch — descendants do not contain the folder segment).

### 3.4 `FrontendRouting/HideToggle.feature` — Tree A, then toggle, bidirectional after every toggle

- Toggle `folder-a` to `hide=false` via `SetNodeProperties` with `{"hideSegmentInUriPath": false}`:
  - `child-in-folder` resolves to `/folder-a/child`; `/folder-a/child` matches `child-in-folder`; `/child` matches nothing.
  - Projection row for `folder-a` has `hideurisegment = 0`; row for `child-in-folder` has `uripath = "folder-a/child"`.
- Toggle back to `true`:
  - `/child` matches `child-in-folder` again; row `uripath` is `"child"`.
- Toggle with same value as current — projection rows untouched (assert by table-equality before and after).

### 3.5 `FrontendRouting/HideToggleOnCreate.feature`

- Create a folder with explicit `hideSegmentInUriPath: false`, then add a child. Child resolves to `/folder-a/child` (no toggle path involved — pure `hideUriSegmentForInsert` + standard core insert).
- Then `SetNodeProperties` → toggle to `true`. Child resolves to `/child`. (This is the "insert path then toggle path" sequence.)

### 3.6 `FrontendRouting/MoveNode.feature`

- Move `child-in-folder` from `folder-a` to `sibling-of-folder` via `MoveNodeAggregate`. Resolves to `/sibling/child`; `/sibling/child` matches it.
- Create a second transparent folder `folder-b` under site; move `child-in-folder` there. Resolves to `/child`; `/child` matches `child-in-folder` (transparency preserved across move; exercises `buildParentUriPath`).
- Move `regular-page` to under `folder-a`. Resolves to `/regular`.

### 3.10 `FrontendRouting/RouteCacheFlush.feature`

- Toggle `folder-a.hideSegmentInUriPath`. Assert that *after the toggle*, a request to the new URL succeeds **immediately** (no replay step in between). If route cache had not been flushed, the old URL would still match and the new URL would 404. This is the **E2E** form of the assertion — no fake-cache recording needed.
- Counter-scenario: setting an unrelated property does **not** invalidate the descendant URLs (a hot cache continues to serve them). Asserted via cache-warm reading before and after.

(If the upstream fake-cache recorder is easy to plug in, also assert tag list. ASK before pulling in the trait; falling back to the URL-based assertion is enough.)

## 6. Risks / open questions

### 6.1 Cross-package trait autoload

Reusing `RoutingTrait`/`BrowserTrait`/`ExceptionsTrait` from `Packages/Application/Neos.Neos/Tests/Behavior/Features/Bootstrap/` requires either a `behat.yml.dist` bootstrap path entry OR a `composer.json` `autoload-dev` mapping into the upstream tree. Both are within the testsuite's typical envelope, but both are environment-specific. **ASK before editing.**

### 6.2 Route-cache flush — recording vs. E2E

Plan defaults to the E2E form (post-toggle URL behavior) which is purer and doesn't depend on which trait wires the fake `RouterCachingService` upstream. If we later want the "tag list" assertion (`ContentCacheTrait` or a routing-specific one), pull it in only at that point.

### 6.3 `dimensionspacepoint` JSON in step parameters

`Dimensions.feature` uses bare-JSON in `the node X in dimension "..."` calls. Strings need double-quoting (`"{\"language\":\"en\"}"`). Lift verbatim from upstream features — do not hand-write.

### 6.4 Defense A coverage of `CreateNodeVariant`

The command-hook's `checkVariant` path must actually intercept `CreateNodeVariant` in Neos 9. Confirm at implementation time by writing 3.14 first as a smoke test — if the command does not flow through `UriCollisionCommandHook`, re-scope 3.14 to whichever command path actually carries variants.

### 6.5 Replay + cache (bug #4253 surface)

`Basic.feature:98-106` explicitly warns about cache-survival masking replay correctness. Our 3.8 replay scenario must not rely on cache state between scenarios — use a fresh CR for the replay assertion. The testsuite's per-scenario reset should do this; verify in commit 3.

### 6.6 `node1` is the conventional site nodeName

`A site exists for node name "node1"` is the upstream convention (`Basic.feature:54`). Keep it across our features for greppability with the upstream tests.
