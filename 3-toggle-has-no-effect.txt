│ Fix: hideSegmentInUriPath Toggle Has No Effect on Stored URI Paths           │
│                                                                              │
│ Context                                                                      │
│                                                                              │
│ Sandstorm.NodeTypes.Folder makes folder nodes "transparent" in URLs via a    │
│ hideSegmentInUriPath boolean property (default true). When this property is  │
│ changed to false (show segment), the projection is never updated — not even  │
│ after a full replay. The bug also affects uriPathSegment changes on          │
│ transparent folder nodes (currently silently ignored).                       │
│                                                                              │
│ Root cause — three interlinked problems in DocumentUriPathProjection.php:    │
│                                                                              │
│ 1. whenNodePropertiesWereSet() ignores hideSegmentInUriPath (lines 578–585   │
│ early-return). The handler watches only uriPathSegment, targetMode, and      │
│ target. Changes to hideSegmentInUriPath are silently dropped.                │
│ 2. generateUriPath() (line 315) and generateParentUriPath() (line 348)       │
│ always skip folder-type nodes regardless of the hideSegmentInUriPath value.  │
│ They check isOfType('…:Document.Folder') unconditionally, so making a folder │
│  opaque has no effect on how child paths are built.                          │
│ 3. uriPathSegment changes on folder nodes are skipped (lines 619–622): even  │
│ if the user renames a folder, the folder's own uriPath is never updated.     │
│ (For transparent folders, children are unaffected — but the folder row       │
│ itself is wrong.)                                                            │
│                                                                              │
│ How the projection stores folder paths:                                      │
│ Transparent folder: uriPath = "my-folder" (stores its own segment, but       │
│ generateUriPath() skips it for children → children get "child" not           │
│ "my-folder/child").                                                          │
│ This means toggling hideSegmentInUriPath must rewrite all descendant paths   │
│ in the DB.                                                                   │
