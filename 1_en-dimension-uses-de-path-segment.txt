⏺ I've identified the bug and written the plan. Here's the summary:

  Bug Analysis

  The bug is in copyVariants() in DocumentUriPathProjection.php (lines 473–488).
   It has two related problems:

  Bug 1 — Folder nodes: When copying a folder to en, $uriPath = 
  $parentNode->getUriPath() is computed but $targetNode->withUriPath(...) is
  never called. The folder variant in en keeps the de dimension's raw uriPath
  unchanged.

  Bug 2 — Child pages of folders: The code naively prepends
  $parentNode->getUriPath() (which for a folder would be e.g. "my-folder") to
  the child's segment. This re-introduces the folder's path segment into the
  child page's URL in en, producing "my-folder/child-page" instead of
  "child-page".

  Fix

  Replace the entire if/else folder-check block with a single call to the
  already-existing generateUriPath() method, which already correctly skips
  folder ancestors:

  if ($parentNode !== null) {
      $uriPathSegment = basename($sourceNode->getUriPath()) ?:
  $sourceNode->getUriPath();
      $uriPath = $this->generateUriPath($uriPathSegment, $parentNode,
  $interdimensionalSibling->dimensionSpacePoint);
      $targetNode = $targetNode->withUriPath($uriPath);
  }

  After applying the fix, a projection replay is needed to correct existing
  broken records.
