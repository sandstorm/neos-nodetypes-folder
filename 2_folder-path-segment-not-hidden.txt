│                                                                              │
│ Fix: Folder Path Segment Not Hidden in Secondary Dimensions                  │
│                                                                              │
│ Context                                                                      │
│                                                                              │
│ Sandstorm.NodeTypes.Folder overrides Neos 9's DocumentUriPathProjection so   │
│ that folder nodes are "transparent" in URLs — their uriPathSegment is        │
│ excluded from child pages' URL paths. This works correctly when nodes are    │
│ first created in the primary dimension (de), but breaks when a content       │
│ variant is created in a secondary dimension (en).                            │
│                                                                              │
│ Root cause: The copyVariants() method (called by all three                   │
│ whenNode*VariantWasCreated handlers) has two related bugs:                   │
│                                                                              │
│ 1. Folder nodes: The computed $uriPath = $parentNode->getUriPath() is never  │
│ applied via $targetNode->withUriPath(...). So the folder variant in en is    │
│ stored with the source dimension's (de) uriPath unchanged.                   │
│ 2. Non-folder children of a folder: The code naively prepends                │
│ $parentNode->getUriPath() (the folder's stored uriPath, e.g. "my-folder") to │
│  the child's last segment. This re-introduces the folder segment into the    │
│ child's URL in en, producing e.g. "my-folder/child-page" instead of          │
│ "child-page".                                                                │
│                                                                              │
│ The generateUriPath() method already exists and correctly skips folder       │
│ ancestors when building a URI. The copyVariants() method should use it       │
│ instead of manual concatenation.                                             │
│                                                                              │
│ Critical Files                                                               │
│                                                                              │
│ - Bug location: app/Packages/Plugins/Sandstorm.NodeTypes.Folder/Classes/Fron │
│ tendRouting/Projection/DocumentUriPathProjection.php                         │
│   - copyVariants(): lines 443–493 (the if/else block inside the $parentNode  │
│ !== null check, lines 473–488)                                               │
│   - generateUriPath(): lines 303–334 (reuse this)                            │
│                                                                              │

│ Why this works:                                                              │
│ - basename($sourceNode->getUriPath()) extracts the raw last segment (e.g.    │
│ "child-page" from "child-page", or "my-folder" from "my-folder")             │
│ - generateUriPath() walks up from $parentNode in the target dimension,       │

│ same logic used when nodes are first created                                 │
│ - For a folder node, generateUriPath("my-folder", siteRoot_en, en) produces  │
│ "my-folder" (correct, consistent with de)                                    │
│ - For a child page under a folder, generateUriPath("child-page", folder_en,  │
│ en) skips the folder and produces "child-page" (correct, folder stays        │
│ transparent)                                                                 │
│ - $targetNode = $targetNode->withUriPath($uriPath) is now always called      │
│ (both folder and non-folder fix)                                             │
